# Board Brief Builder

把零散业务材料压成高管能直接讲出去的 brief。

## 适用场景

- 董事会汇报
- 月度 / 季度经营复盘
- 对上管理层 update
- 用户说“准备汇报”“给我一版 briefing”“这堆材料怎么讲”

## 工作流

1. 先判断这次 brief 要解决的核心问题，只保留一个主问题。
2. 从材料里提炼：
   - 发生了什么
   - 为什么重要
   - 该怎么说
   - 可能被追问什么
3. 把无关背景、枝节数据、重复事实删掉。
4. 输出一版可直接讲的高密度结构。

## 默认输出

```text
【这次汇报要解决什么】

【结论】
- 1句总判断

【三点主线】
- 主线1
- 主线2
- 主线3

【关键证据】
- 只保留最有说服力的事实 / 数字 / 对比

【建议表达方式】
- 对上怎么讲

【可能被追问的问题】
- Q1
- Q2
- Q3
```

## 规则

- 先“能讲出去”，再追求信息齐全
- 不把内部假设说成确定事实
- 一页里不要超过 3 条主线
- 数字只保留有判断价值的
- 如果材料不足，最多补 1 个关键追问
