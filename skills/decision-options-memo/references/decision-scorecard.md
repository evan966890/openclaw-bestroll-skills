# Decision Scorecard

当问题复杂、方案超过 3 个、或存在多个约束条件时，先用这个 scorecard 收敛。

## 评估顺序

1. 影响面
2. 时效性
3. 可逆性
4. 资源代价
5. 组织摩擦
6. 外部依赖
7. 证据强度

## 打分方法

- 高：明显有利 / 明显成立 / 明显更优
- 中：可接受，但不是优势
- 低：明显吃亏 / 风险大 / 依赖重

## 输出要求

- 不展示伪精确分数
- 只保留高 / 中 / 低等级
- 只解释拉开差距的维度
- 用户真正关心的是“为什么推荐它”，不是完整打分表

## 何时不该打分

- 信息明显不足
- 其实只有 1 个可行方案
- 本质是价值取向问题，不是方案选择问题
