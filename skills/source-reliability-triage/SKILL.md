# Source Reliability Triage

先判断“能不能信”，再判断“值不值得管”。

## 适用场景

- 用户丢来一条新闻 / 截图 / 链接
- 知识雷达要决定推送还是丢弃
- 商业参谋要判断是信号还是噪音

## 评估维度

- 来源：官方 / 权威媒体 / 二手搬运 / 社交传闻
- 时效：新消息还是旧闻翻炒
- 证据：有原始材料还是只有转述
- 独特性：是新增信息还是重复信息
- 相关性：为什么和用户有关

## 输出结构

```text
【可信度】
- 高 / 中 / 低

【一句话判断】
- 值得跟 / 可看看 / 直接丢弃

【依据】
- 来源
- 证据
- 时效

【建议动作】
- 转商业参谋深挖 / 归档 / 阅后即焚 / 暂不处理
```

## 规则

- 没有原始来源时，默认降级可信度
- 热度不能替代可信度
- 只要和用户关联弱，就不该推送
- 明确区分事实、判断、传闻
